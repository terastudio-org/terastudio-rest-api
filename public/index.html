API Endpoints

Below are the available endpoints with validation and error handling:

POST
/api/users/register
Register a new user with validation

Body Parameters:

username string required Min length: 3, Max length: 30
email string required Must be a valid email format
password string required Min length: 6
age number Min: 13, Max: 120
Example Request:
curl -X POST http://localhost:3000/api/users/register \
-H "Content-Type: application/json" \
-d '{"username": "johndoe", "email": "john@example.com", "password": "password123", "age": 25}'
GET
/api/users/get
Get a paginated list of users with search functionality

Query Parameters:

page number Default: 1, Min: 1
limit number Default: 10, Min: 1, Max: 100
search string Max length: 50
Example Request:
curl -X GET "http://localhost:3000/api/users/get?page=1&limit=10&search=john"
GET
/api/openapi.json
Get complete API specification with all available endpoints
Example Request:
curl -X GET http://localhost:3000/api/openapi.json

Response Format

All API responses follow a standardized format:

Successful Response:

{
  "statusCode": 200,
  "success": true,
  "timestamp": "2023-12-07T10:00:00.000Z",
  "attribution": "@terastudio-org",
  "data": { /* response data */ },
  "message": "Operation successful"
}
Error Response:

{
  "statusCode": 400,
  "success": false,
  "message": "Validation failed: username is required",
  "code": "VALIDATION_ERROR"
}

Validation Examples

The API includes comprehensive validation with detailed error messages:

Valid Request:
{
  "username": "validuser",
  "email": "user@example.com",
  "password": "password123"
}

Invalid Request (returns 400):
{
  "username": "jo", // Too short
  "email": "invalid-email", // Invalid format
  "password": "123" // Too short
}

Error Response:
{
  "statusCode": 400,
  "success": false,
  "message": "Validation failed: username must be at least 3 characters long, email format is invalid, password must be at least 6 characters long",
  "code": "VALIDATION_ERROR"
}

Error Handling

The API handles different types of errors with appropriate status codes:

  • 400 Bad Request - Validation errors, missing required parameters
  • 404 Not Found - Endpoints that don't exist
  • 413 Request Entity Too Large - File uploads that exceed size limits
  • 500 Internal Server Error - Unexpected server errors
